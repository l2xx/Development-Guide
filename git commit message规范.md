## 格式

```
<type>: <subject>
<BLANK LINE>
<body>
```

### type

- **feat**: 新功能
- **fix**: 修复Bug
- **docs**: 文档
- **style**: 格式，不影响代码运行的改动 (white-space, formatting, missing semi-colons, etc)
- **refactor**: 重构 (既不是新功能也不是修复Bug的代码改动)
- **perf**: 提升性能的改动
- **test**: 测试相关
- **chore**: 构建工具或辅助工具的变动

### subject

简短描述，不超过50个字符

### body

详细描述，可选。
